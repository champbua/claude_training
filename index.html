Foundation

What is Claude?

Claude is Anthropic's AI assistant, designed to be helpful, harmless, and honest. Unlike a search engine, Claude understands context, reasons through problems, and produces nuanced, human-quality responses.

🔬 Built by Anthropic

Anthropic was founded in 2021 by former OpenAI researchers focused on AI safety research. Claude is trained using Constitutional AI — a technique designed to make AI more steerable and aligned.

🧠 What Claude can do

  • Write, edit, and summarise text
  • Analyse data, code, and documents
  • Research and synthesise information
  • Reason through multi-step problems
  • Have extended, contextual conversations
Key insight: Claude's strength is reasoning, not just retrieval. It can follow complex instructions, maintain context across a long conversation, and adapt its style to your needs.

Platforms

Web App vs Desktop App

Claude runs in your browser at claude.ai and as a native Desktop app for Mac and Windows. Both share your account and conversations, but have distinct capabilities.

🌐

Web App (claude.ai)

  • No install required — works on any OS
  • Full Chat + Cowork experience
  • Access from any device
  • Cowork plugins available
  • Upload files up to 10 MB per file
  • Limited local system access
🖥️

Desktop App (Mac / Windows)

  • Runs natively — faster UI response
  • Can see your screen (with permission)
  • Understands what app you're working in
  • Global hotkey to open Claude anywhere
  • Works offline for some features
  • Better for file-heavy workflows

Desktop-Exclusive Features

🖱️ Screen awareness ▼
The desktop app can analyse what's on your screen in real time — you can ask "explain what I'm looking at" and Claude will read the active window, error message, or diagram without you having to copy-paste anything.
⌨️ Global hotkey ▼
Open a Claude prompt overlay from any app using a keyboard shortcut (configurable in settings). Great for quick lookups while you're deep in another tool — no tab switching needed.
📁 Local file access ▼
On desktop, you can drag-drop files directly from Finder/Explorer into the chat window, and the desktop app can read local file paths more naturally in supported workflows.
🔧 IDE Extensions (VS Code, JetBrains) ▼
Claude Code integrates directly into VS Code and JetBrains IDEs, giving you in-editor AI assistance — inline suggestions, code explanations, and one-click refactors — without leaving your editor.
✅

Recommendation: Use the web app when you're on the go or need Cowork. Use the desktop app for focused work sessions where screen context or hotkeys will save you time.

Prompting

Prompt Tips & Tricks

The quality of your prompt is the single biggest lever you have over Claude's output. These patterns consistently produce better results.

❌ vs ✅ — Common Patterns

❌ Vague prompt
Write something about our product.
No audience, no goal, no format, no constraints → generic output.
✅ Structured prompt
Write a 3-paragraph email to a first-time e-commerce customer who just placed their first order. Tone: warm and confident. Include: order confirmation, shipping timeline (3–5 days), and a 10% discount for their next order. Max 150 words.
Audience + goal + format + constraints + tone = much better output.

The CRAFT Framework

C — Context

Who is the audience? What's the situation? Give Claude the background it needs.

"You are helping a junior marketing analyst..."

R — Role

Tell Claude what role to play. This shifts the voice, knowledge, and style of the output.

"Act as a senior UX researcher..."

A — Action

State the task clearly with a strong verb: write, analyse, summarise, compare, draft.

"Compare these two pricing strategies..."

F — Format

Specify output structure: bullet list, table, JSON, markdown, numbered steps.

"Output as a table with 3 columns..."

T — Tone & Constraints

Set tone (professional, casual, concise) and add hard limits (max words, avoid jargon, no bullet points).

"Formal tone, max 200 words, no technical jargon..."

Power Techniques

🔗 Chain-of-thought: "Think step by step" ▼
Adding "Think step by step" or "Reason through this before answering" prompts Claude to break down its reasoning — dramatically improving accuracy on logical, maths, or analytical tasks. Claude reasons internally before writing the final answer.
📝 Few-shot examples ▼
Show Claude an example of the input/output pair you want. For instance:

Input: "Product arrived late." → Output: "We're sorry for the delay — your order was held up in transit. Here's a 10% voucher for your trouble: [CODE]"

Then ask Claude to follow the same pattern for your new cases.
🔄 Iterative refinement ▼
Don't expect perfection on the first try. Follow up with refinements: "Make it shorter," "Less formal," "Add a call-to-action at the end," "Rewrite the second paragraph." Claude retains full context throughout a conversation.
📌 System prompts / Custom instructions ▼
In Projects (claude.ai), you can set a persistent system prompt that applies to every message in that project. Use this to define a persona, set rules ("always respond in Thai"), or provide standing context about your company — without repeating it every time.
🎯 Negative constraints ▼
Tell Claude what NOT to do: "Don't use marketing buzzwords," "Don't add bullet points," "Don't apologise in your response." Negative constraints are often more effective than positive ones for controlling style.

Tokens & Context Window

Understanding tokens is the key to understanding Claude's limits, costs, and why long conversations sometimes "forget" earlier messages.

🔤 What is a token?

Tokens are the units Claude "reads" in. Roughly:

  • ~4 characters = 1 token
  • ~0.75 words = 1 token
  • 1 page of text ≈ 500–750 tokens
  • 1 typical API message ≈ 200–800 tokens

📦 What is a context window?

The context window is how much Claude can "see" at once — everything in the window is visible; everything outside is forgotten.

  • Includes: system prompt + all messages + response
  • Claude Sonnet 4: 200,000 tokens (~150,000 words)
  • When full, older messages are summarised or dropped

Practical Token Tips

🧹 How to manage a long conversation ▼
Start a new conversation when your task shifts significantly. Long threads accumulate tokens fast — older context can interfere with new instructions. Use Projects with a system prompt to maintain persistent context without filling the window.
📄 Uploading large files ▼
Pasting an entire document into a prompt consumes tokens fast. Instead, upload files as attachments — Claude can selectively read the relevant parts without using your conversation tokens for every line.
💰 Cost & tokens (API users) ▼
The API charges per input token (what Claude reads) and output token (what Claude writes). Output tokens cost more. For high-volume tasks, choose the cheapest model that meets quality requirements, keep system prompts lean, and use prompt caching to avoid re-billing repeated context.

Advanced

Advanced Features & Best Practices

These features separate power users from beginners. Learn to use Projects, memory, artefacts, and agentic features effectively.

📁

Projects

Persistent workspaces on claude.ai. Each project has its own:

  • System prompt — standing context
  • Knowledge files — docs Claude can reference
  • Chat history — persists between sessions
  • Plugins — tools active for that project
📎

Artefacts

When Claude generates code, documents, or structured data, it can create an Artefact — a rendered, downloadable output in a side panel. Great for:

  • HTML / React previews
  • Markdown documents
  • SVG diagrams
  • CSV / JSON data
🔁

Agentic Loops

Claude Code and Cowork can run autonomous multi-step loops — planning, executing, checking results, and adjusting. Best used with a clear goal and explicit stop conditions.

⚠️

Always review what an agent will do before approving irreversible actions.

📊

File Analysis

Claude can read and reason over uploaded files:

  • PDFs, Word docs, spreadsheets
  • Images (screenshots, charts)
  • Code files of any language
  • Ask questions directly about content

Top 10 Power User Tips

  1. Use Projects for recurring workstreams — set the system prompt once, not every time.
  2. Paste error messages verbatim — Claude diagnoses them better with the exact text.
  3. Ask Claude to ask you questions — "Before answering, ask me any clarifying questions you need."
  4. Request multiple options — "Give me 3 different approaches with pros/cons of each."
  5. Use Claude to improve your own prompts — "Rewrite this prompt to be more effective."
  6. Specify what you don't want — negative constraints are powerful style controls.
  7. Give Claude permission to push back — "Tell me if you think there's a better approach."
  8. Use structured outputs for automation — "Return a JSON object with keys: title, summary, tags."
  9. Set a word limit — Claude naturally writes more than needed; a limit forces concision.
  10. Reference earlier parts of a conversation — "Going back to the analysis you did 3 messages ago..."
